## Change log

### v0.4 — 2026-03-29

**T1/T2 root folder prefixing**
- Applied `NN_` numeric prefix to all root folders for human visual orientation.
- Prefix convention is root-only: internal subfolders, governed object homes, and
  LNS-named files are not prefixed. Agent cold-start paths are declared exclusively
  in `AGENTS.md` and `REPO_PROFILE.md` — agents must derive all paths from those
  sources, never from inference or memory. This isolates T2 (agent navigation) from
  any future T1 reordering.

**Governance consolidation — `01_governance/`**
- Root `standards/` renamed to `01_governance/`. Now the single governance library.
- Root `framework/` collapsed into `01_governance/framework/`.
- New `01_governance/playbooks/` subfolder added for operating guides and how-to
  references.
- Root `decisions/` collapsed into `01_governance/decisions/`. Decisions are governance
  artifacts — they belong in the governance library, not as a peer root folder.
- `decision-register.md` and `exception-register.md` moved into
  `01_governance/decisions/`. Exceptions are governance decisions; the decision register
  is the authority index for all ADRs.

**`02_templates/` positioned as bridge layer**
- Retained as a standalone root folder. Templates are downstream of governance
  (they operationalize it) and upstream of execution (agents reach for them when
  creating new objects). Position between `01_governance/` and `03_portfolios/` is
  intentional and correct.

**Cherry-pick CP1 — YAML companion files**
- `entity.yaml` added alongside the primary `.md` governing file at every portfolio,
  program, and project home. `packet.yaml` added at every task-packet home.
- `.md` files are retained for human readability and agent context.
- `entity.yaml` / `packet.yaml` are the machine-readable source of truth for
  structured fields: status, dates, assignees, dependencies, labels.

**Cherry-pick CP2 — JSON derived registers in `06_state/`**
- Root `registers/` folder retired. Entity registers (portfolio, program, project,
  task) are now fully derived from `entity.yaml` / `packet.yaml` scans.
- `06_state/registers/` holds derived JSON outputs regenerated by `rebuild-state.ps1`.
- Agents query JSON registers; humans do not maintain them manually.

**Cherry-pick CP3 — `06_state/` machine-managed zone**
- New root folder `06_state/` declared as agent-written territory.
- `06_state/inputs/` holds human-authored YAML for cross-cutting concerns that cannot
  be derived from entity files alone: risks, dependencies, benefits, artifacts.
- `06_state/graph/` holds the derived dependency graph.
- `06_state/audit/` holds the immutable log of agent state changes.
- `06_state/README.md` declares the input vs. output contract for the zone.

**Register redistribution (retiring root `registers/`)**
- `portfolio-register.md` — retired; derived from `entity.yaml`.
- `program-register.md` — retired; derived from `entity.yaml`.
- `project-register.md` — retired; derived from `entity.yaml`.
- `task-register.md` — retired; derived from `entity.yaml`.
- `decision-register.md` — moved to `01_governance/decisions/`.
- `exception-register.md` — moved to `01_governance/decisions/`.
- `risk-register.md` — moved to `06_state/inputs/` as `risk-register.yaml`.
- `dependency-register.md` — moved to `06_state/inputs/` as `dependency-register.yaml`.
- `benefit-register.md` — moved to `06_state/inputs/` as `benefit-register.yaml`.
- `artifact-register.md` — moved to `06_state/inputs/` as `artifact-register.yaml`.
- `repo-register.md` — moved to `04_docs/` as a repo-level reference document.

**Root folder renumbering**
- Final root sequence: 9 folders (down from 11). `decisions/` and `registers/` retired
  as standalone roots; absorbed into `01_governance/` and `06_state/` respectively.

## Alignment notes

- `03_portfolios/` is the canonical top governed business container.
- `programs/` live under portfolios. `projects/` live under programs.
  `task-packets/` live under projects.
- Primary governing `.md` files live at the root of each governed object home,
  accompanied by a sibling `entity.yaml` (or `packet.yaml` for task packets).
- `README.md` files are present at each hierarchy layer including `task-packets/`.
- Standalone `PORT`, `PROG`, and `PROJ` identity prefixes are retired; replaced by
  `P3-PORT`, `P3-PROG`, and `P3-PROJ` per LNS v0.4 §9.
- Durable decision examples use `ADR-DECN` per LNS v0.4 §8.2. Architecture decision
  records use `ADR-ARCH` per LNS v0.4 §8.2.
- **`ADR-DECN` IDs are globally sequential across all decision scopes.** Scope is
  recorded in front matter (`scope_level: Enterprise | Portfolio | Program | Project`),
  not in the ID or folder path. The example IDs in this tree (`0001` at repo-wide,
  `0002` at portfolio, `0003` at program, `0004` at project) are illustrative of the
  continuous sequence — they are not scope-partitioned ranges.
- Task packet examples use the full LNS task formula.
- Template, pattern, and framework examples use LNS-compliant `STD-*` families.
- Conventional navigation and control-surface files (`README.md`, `START_HERE.md`,
  `AGENTS.md`, `CLAUDE.md`, register files, script entrypoints) are retained as
  operational navigation surfaces and are not renamed as P3 identity documents.
- **Root folder prefix convention:** `NN_` numeric prefixes apply to root folders only.
  Internal subfolders, governed object homes, and LNS-named files are not prefixed.
- **Agent path isolation:** all agent-navigable paths are declared in `AGENTS.md` and
  `REPO_PROFILE.md`. Agents must read paths from those sources — never infer from folder
  names or numbering. This makes T1 reordering safe without breaking agent navigation.
- **`06_state/` contract:** `06_state/inputs/` is human-authored YAML (cross-cutting
  registers that cannot be derived). `06_state/registers/`, `06_state/graph/`, and
  `06_state/audit/` are machine-written outputs. Never edit the machine-written zone
  directly — use `07_scripts/rebuild-state.ps1`.
- **Sequencing note:** the `framework/` file rename (`P3-GOV-0001` → `STD-REFR-0001`)
  must be deferred until after `README.md`, `AGENTS.md`, and `START_HERE.md` have been
  updated to reference the new path. Renaming before the control surfaces are updated
  will produce dead references in agent cold-start paths.
