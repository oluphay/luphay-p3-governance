This Pull Request (PR) formalizes the introduction of the PLAY (Playbook) class and updates the ID logic to prioritize technical accuracy for external standards. This change ensures that Luphay’s governance system can distinguish between rigid routines and adaptive, scenario-based responses while maintaining a precise link to global regulatory IDs.PR Summary: Refactoring the STD FamilyKey ChangesIntroduced PLAY Class: Created a new category for situational, branching response patterns (e.g., incident response) to separate them from linear procedures (PROC).Variable-Width ID Logic: Updated the naming formula to allow for full-length external standard IDs (e.g., 27001 or 80053) instead of trancated 4-digit serials.Boundary Clarification: Detailed the "Decision Point" logic used to choose between PROC, PLAY, and GUID.Precedence Model: Established the internal hierarchy: CONS > POLI > PROC / PLAY > GUID.Updated Naming Standard5. STD — StandardsPurpose: The STD family governs normative, reference, and adopted standards used to direct how Luphay operates, decides, documents, and assures quality. It separates binding authority from operational execution, recommended practice, reusable scaffolds, and static reference material.Formula:PlaintextSTD-[CLASS]-[ID]__[slug]__vX.Y.md
5.1 Internal Class Definitions and BoundariesClassTypeBoundary DefinitionCONSConstitutionFoundational governance principles and system-defining rules.POLIPolicyMandatory requirements and non-optional constraints.PROCProcedureCanonical step-by-step methods for routine, stable workflows.PLAYPlaybookScenario-based response patterns for incidents or coordinated actions.GUIDGuidelineAdvisory heuristics and advice where judgment is permitted.TEMPTemplateReusable master documents designed for structural consistency.REFRReferenceStable factual or taxonomic material used for lookup/citation.5.2 Decision Logic: PROC vs. PLAY vs. GUIDWhen selecting a class, apply the following operational logic:Choose PROC if: The artifact answers “How is this routine process performed?” It should be primarily step-by-step with a fixed path and zero to minimal discretion.Choose PLAY if: The artifact answers “What do we do when this situation happens?” It is triggered by an event and includes branches, coordination across roles, or escalation paths.Choose GUID if: The artifact is advisory. It answers “What does good practice look like?” and improves quality without imposing a hard mandate.5.3 Special Rules for IDs and External StandardsFor EXTR (External) and REGL (Regulatory) classes, technical accuracy of the ID is prioritized over fixed-width formatting.Internal IDs: Continue using a fixed 4-digit serial (e.g., 0001, 1102).External IDs: Use the full actual standard number (e.g., 27001 for ISO 27001 or 2016679 for EU GDPR).Slugs: For external sources, always start the slug with the issuing body (e.g., iso-, nist-, pa-).Version: Use the publication year (e.g., v2022) or specific source version.5.4 Implementation ExamplesPlaintext# Internal Standards (Fixed 4-digit ID)
